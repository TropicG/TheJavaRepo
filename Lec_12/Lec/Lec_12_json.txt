* Формати за обмен на данни

    Форматите за обмен на данни са зависими от използваната архитектура или протокол
    # Това означава, че начинът, по който е изградена системата, диктува какви файлови формати (примерно XML или JSON) могат да се използват

    Уеб услуги, базирани на SOAP (Simple Object Access Protocol) - поддържат само XML

    Уеб услугите, базирани на REST не налага използването на конкретен формат за обмен на данни, но най-популярният е JSON
    # Други формати като XML, YAML и други могат да се използват 

* JavaScript Object Notation (JSON)

    JSON е един от най-популярните формати за обмен на данни, като той произлиза от JavaScript 

    Използва се за предаване на структурирани данни по мрежова комуникация
    # Главно този формат служи като транспорт за инфромация между различни системи през интернета

    JSON намира широко приложение в RESTful уеб услугите за представяне на данни

* JSON - характеристики

    JSON има следните характеристики:
    - Web friendly - тривиално парсване (за разлика от XML)
    # един json позволява на софтуера изключително лесно и бързо да разчетеданните, това прави приложенията по-бързи
    - Кратък и интуитивен
    - Разбираем едновременно от човек и машина

* Типове данни, които JSON поддръжа 

    Списък с данните, които JSON поддържа:
    - Boolean: true / false
    - Number: 42, 3.14156, -1
    # В JSON няма разлика между цели числа, дробни числа, всички числа се борят като число
    - String: "foo", "bar"
    # Всякакъв текст трябва задължително да бъде ограден в двойни кавички
    - Array: ["John", "Anna", "Peter"]
    # Списъка на елементите, трябва да бъде подреден последователно и да е обграден с []
    - Обекти: {"name": "John", "age": 30}
    # ова е съвкупност от двойки key-value и са обозначени с {}

    Правила, които да се спазва и какво не при писане на JSON:
    - ппц може да имаш null или {}
    - не можеш да пишеш функции или дати или undefined

* Библиотеката google/gson

    Това е библиотека за работа с JSON, наречена google/gson, предназначена за програмисти, използващи езика Java

    Основната задача на тази библиотека е да предостави лесен механизъм за преобразуване от Java обект към JSON и обратно
    # Това може да се случи с методите .toJson() и .fromJson()

    Библиотеката не изисква добавяне на анотации или прекомпилиране, това означава, че директно може да се използват Java обекти

    Поддържа Builder, който позволява конфигуриране на различни опции
    # Това е design pattern
