След прилагането на type erasure, сигнатурите на методите в двата класа имат различна сигнатура за JVM:
- В Box: setValue(Object value)
- В BoxOfInt: setValue(Integer value)
Те имат различни параметри, следователно различни структури, тогава технически BoxOfInt метода не пренаписва метода в Box.
Това би счупило полиморфизма, защото извикването на setValue през референцията от тип Box не би намерил съответния метод в BoxOfInt 
За да се справи с този проблем комиплатора генерира синтетичен (невидим в кода) метод в класа BoxOfInt, този метод се нарича мостов метод 
Пример: 
	
	// този код се генерира автоматично от комиплатора в BoxOfInt.class
	public void setValue(Object value) {
	
		// прави правилния cast и извиква истинския метод
		setValue((Integer) value);
	}
