Важни неща, които трябва да се запомнят:

- Java е обектно-ориентиран език за програмиране и е популярен за уеб приложениея, мобилни и корпоративни приложение
- Java компилатора, компилира Java source код (завършващ на .java) в байтов код (завършва на .class), който след това се изпълнява от JVM (Java Virtual Machine)
- JVM-a е реално софтуерен слой, който позволвяа изпълнението на Java програми, независимо от ОС, като интерпретира байткода
- JDK-то съдържа компилатора, библиотеките и средата за изпълнение на Java програми, използва се за разработване на Java приложения
- JRE съдържа само в себе си библиотеките и средата за изпълнение на Java програми, но не и компилатор

- Java е със C/C++ синтаксис като е статично типизиран, тоест всяка променлива има зададен тип и се проверява по време на компилация
- Java е мултипарадигмен (поддържа ООП както и елементи от функционалното програмирнае като lambda изрази и stream API-то)
- JVM-a съдържа система за автоматично управление на паметта, която се намира в heap-a и се нарича Garbage Collector
- Java езикът е много portable, защото може да се пусне на всяка платформа, стига да има инсталиран JVM

- Когато се стартира една Java програма, JVM търси public static void main(String[] args) метода 

- Процес на изпълнение на едно Java приложение:
1) Java код се компилира във байт код (това са файловете със разширения .class)
2) Този .class файл се подава на JVM-a
3) JVM-a интерпретира байткода и го компилира на машинен език, който машината разбира
# JVM-a реално прави JIT (Just in Time) компилация, реално докато интерпретира този байткод го компилира на машинен код

- Следните неща, които JVM-a прави докато работи едно Java приложение:
1) Стартира нишки и да ги кординира използвайки JVM Scheduler-a
2) Да комуникира с ОС, защото JVM е като софтуерен мост между Java програмата и OС
3) JVM-a зарежда класове в паметта като намира .class файловете, когато даден клас бъде извикан или използван в кода
4) JVM поддържа автоматичното изчистване на динамично заделената памет чрез Garbage Collector-a

- Създаването на една променлива в Java се извършва от две стъпки: декларация (даване на името на променливата) и инициализация (подаване на стойност към променливата)
- В Java променливите се делят на два типа: референтни и примитивни

- Примитивните типове съдържат в себе си самата стойност на променливата, примери за това са: boolean, int, double, ...
- Референтните типове съдържат референция към мястото в паметта, където е записан обекта: пример може да бъде всеки създаден клас 

- Wrapper типовете са обекти в Java, които обвиват стойност на примитивна стойност в обект, примери Integer, Double, Boolean
# Използват се, защото в декларирането на колекции, не можем да използваме референтни типове 

- Autoboxing е когато Java автоматично присвоява примитивна стойност към обект от Wrapper за същата стойност (Character a = 'a')
- Uboxing е когато Java автоматично взима стойноста от Wrapper обект и я присвоява към същия примитивен тип (Integer num1 = Integer.valueOf(2); int num2 = num1)

- Низовете в Java са референтен тип, а не примитивен, защото съдържат референция към низ-а в паметта
- Важно нещо за низовете е, че те са immutable и веднъж създаден не може да се промени, при промяна Java създава изцяло нов низ и обекта сочи към него

- String pool-a е специална област в JVM-a, която служи за памет на низове:
# Когато създадем нов string literal JVM проверява дали съществува в string pool-a, ако съществува връща референция, ако ли не добавя този литерал в string pool-a 

- Heap-a е основната динамична памет в Java, където се съхраняват всичко обекти създадени чрез new, дори ако създадем String чрез new оператора ще отиде в heap-a като пропусне string pool-a
